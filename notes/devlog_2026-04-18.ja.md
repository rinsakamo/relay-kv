## メモ

今回のポイントは、same-step apply を無理に追わず、repo の現状に合わせて next-step apply に切り替えたこと。
研究プロトタイプとしては、最小差分で「実際に hook で差し替えた」ことに加え、「generation divergence がどの step で現れるか」「prompt style ごとに divergence lag がどう違うか」まで確認できたのが大きい。

Summary:
- Investigated RelayKV next-step apply behavior across gate policies (strict, medium, relaxed)
- Identified that strict gate produced zero events across all prompt types
- Medium gate worked for seq_len=1024 but failed for seq_len=2048
- Root cause: block id mismatch (1024 uses block 3, 2048 uses block 6/7)
- Introduced medium_2048 gate policy: (6, 7, 14, 15)

Key Findings:

[1024 / medium]
- prose: lag 2
- repetitive: lag 3
- structured: lag 1

[2048 / medium_2048]
- prose: lag 0
- repetitive: lag 1
- structured: lag 0

Interpretation:
- Longer context shifts dominant attention blocks
- Correct block selection drastically affects intervention visibility
- Repetitive prompts remain more stable (longer lag)
- Structured and prose respond more immediately under correct block routing

Top5 Analysis (medium):
- repetitive: partial overlap (ranking shift)
- prose/structured: low overlap (candidate set shift)

Conclusion:
- Gate policy must be seq_len-aware
- Block migration is primary driver, not threshold
- medium_2048 restores functionality without relaxing threshold

Next Steps:
- Compare top5 logits for 2048
- Generalize gate policy across seq_len
- Possibly dynamic block selection instead of static sets
