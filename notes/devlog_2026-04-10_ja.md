## layer 比較

### 実験条件
- `seq_len = 4096`
- `prompt_type = prose`
- `hot_window = 128, 256`
- `block_size = 128, 256`
- `top_k = 1, 2, 3`
- `layer_idx = 0, 14, 27`

### 見えたこと
3 layer すべてで

- coverage_ratio が上がるほど mean_abs_diff が下がる

ことを確認した。

ただし、絶対誤差の大きさは layer ごとにかなり違った。

### 代表的な傾向
- `layer 14` が最も近似しやすい
- `layer 0` がその次
- `layer 27` が明確に難しい

### 解釈
- coverage-driven trend は layer をまたいでも保たれる
- ただし絶対的な approximation difficulty は layer 依存が強い
- 深い layer はやはり難しい
- ただし浅い→中間→深い が単純単調ではなく、今回の slice では `layer 14` が `layer 0` より易しかった

### 作成した図
- `relaykv_layer_compare_4096_prose_v2.png`

### 図の意味
- x軸: `coverage_ratio`
- y軸: `mean_abs_diff`
- 系列: `layer 0 / 14 / 27`

この図で、

- layer が違っても coverage trend は残る
- でも難しさはかなり違う

ことが見やすくなった。

## scoring variant の試行

### まず試したもの
- `mean_only`
- `mean_plus_norm`
- `mean_plus_vnorm`

### 結果
`mean_plus_norm` も `mean_plus_vnorm` も、今回の条件では retrieval behavior を変えなかった。

### 解釈
- `query × k_mean` の項がかなり支配的
- norm 系の軽い補正では順位が変わらない
- 少なくとも今回の setup では、norm augmentation は有効ではなかった

---

## headwise 系 variant

### 追加したもの
- `headwise_max_mean`

### 狙い
- 単純平均ではなく、head ごとの強い反応を残す scoring を試したかった
- norm 系よりは別系統の変更になると考えた

### sweep 上の観測
一時的には、ある 4096 条件で `mean_only` より少し良く見えるケースがあった。

### その後の確認
改善が見えた条件を

- `run_relaykv_pipeline.py`
- `prompt_type=prose`
- `seq_len=4096`
- `hot_window=256`
- `block_size=256`
- `top_k=3`
- `layer_idx=27`

で詳しく確認した。

### pipeline-level inspection の結果
- `mean_only` と `headwise_max_mean` は同じ top block を選択した
- `top_scores` の順位は block id ベースで一致
- `mean_abs_diff` も一致した

### 解釈
- 少なくとも inspected pipeline 条件では、`headwise_max_mean` は `mean_only` に対して retrieval を変えなかった
- 以前 sweep で見えた差は、prompt 生成や実行経路の不整合の可能性が高い
- いったん safest interpretation としては、「軽量な scoring 変更では上位 cold block はかなり安定している」と見るのが自然

## pipeline と sweep の整合確認

### 起きたこと
最初、sweep では `headwise_max_mean` がわずかに改善したように見えた一方で、pipeline では差が出なかった。

### 原因候補
最も怪しかったのは prompt 生成の不一致だった。

- `run_attention_sweep.py` では `prompt_type` を明示していた
- `run_relaykv_pipeline.py` は最初は別の prompt 生成を使っていた

### 対応
- `run_relaykv_pipeline.py` に `prompt_type` を追加
- sweep と同じ prompt 生成ロジックに揃えた
- summary に `prompt_type` と `scoring_variant` を保存するようにした

### 結果
prompt 生成を揃えて再比較したが、inspected pipeline 条件では `mean_only` と `headwise_max_mean` は同じ block を選択した。

### 理解
- variant 比較を信頼できるものにするには、sweep と pipeline で prompt 生成ロジックを揃える必要がある
- 今回の discrepancy は scoring そのものより、実行経路の差による可能性が高かった

## 今日の到達点

RelayKV の experimental story として、さらに次のことが言いやすくなった。

- coverage-driven trend は layer をまたいでも保たれる
- absolute difficulty は layer ごとにかなり違う
- prompt を揃えた pipeline inspection では、軽量な scoring 変更は block selection を変えなかった
- したがって、次の scoring 設計はもっと構造的に違う方向が必要そう
