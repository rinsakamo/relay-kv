## 実装したもの

### 1. hot / cold split
- `TierManager` に基づいて seq range を分割
- hot KV と cold KV を切り分けた
- cold KV は CPU に退避

### 2. cold cache
- `ColdCache`
- `ColdSegment`

を実 tensor ベースで扱えるようにした

### 3. block 化
- `ColdSegment.to_blocks()` を実装
- cold KV を block 単位に分割
- block ごとの token range と tensor shape を保持

### 4. metadata
- `block_metadata.py` を作成
- 各 block に対して
  - `k_mean`
  - `v_mean`
  - `k_norm`
  - `v_norm`
  を計算

### 5. scoring
- `block_scoring.py` を作成
- query と metadata の簡易内積で score を計算
- `top_k_blocks()` で上位 block を選択

### 6. retrieval
- `block_retrieval.py` を作成
- score 上位 block を実 tensor として回収

### 7. candidate KV
- `candidate_kv.py` を作成
- retrieved blocks を token 順に連結して candidate KV を構成

### 8. working KV
- `working_kv.py` を作成
- candidate KV と hot KV を連結して working KV を構成

### 9. attention comparison
- `attention_compare.py` を作成
- full KV と working KV で attention 出力差を比較できるようにした

## attention comparison の確認

### 小さいケースでの結果

- `full_k_shape = [1, 2, 386, 128]`
- `working_k_shape = [1, 2, 384, 128]`

それでも差分はかなり小さかった。

- `mean_abs_diff = 1.455737219657749e-08`
- `max_abs_diff = 1.1920928955078125e-07`
- `l2_diff = 4.623874758635793e-07`

### 解釈

- cold KV の一部を落としても、selected cold blocks + hot KV だけでかなり full に近い attention 出力が出せた
- prototype path の wiring はかなり正しくできている

## sweep で見えたこと

特に `layer_idx = 27` でかなりきれいな傾向が出た。

### 見えた傾向

- coverage_ratio が上がるほど mean_abs_diff が下がる
- 同じ coverage_ratio なら block_size や top_k が違っても誤差がかなり近い
- hot_window を大きくすると安定しやすい
- seq_len が長くなると難しくなるが、coverage-driven な改善傾向は維持される

### 代表的な理解

- approximation error decreases as candidate coverage increases
- for matched coverage, different structural settings often yield similar errors
- larger hot windows improve stability
- longer contexts remain harder, but follow the same qualitative trend

### prototype-stage の慎重な言い方

今の段階では

- 理論保証
- 一般定理

ではなく、**empirical finding** として扱うのが安全
