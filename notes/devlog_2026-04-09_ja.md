## 4096 を含む追加 sweep

### 条件
- `seq_len = 1024, 2048, 4096`
- `hot_window = 128, 256`
- `block_size = 128, 256`
- `top_k = 1, 2, 3`
- `layer_idx = 27`

### 見えたこと
4096 でも

- coverage_ratio が上がるほど mean_abs_diff が下がる
- 同じ coverage なら block_size / top_k が違っても近い誤差になる
- coverage-driven trend が維持される

ことを確認できた。

### 代表例
4096 / layer 27 では

- `coverage_ratio = 0.0323` → `mean_abs_diff = 0.04121`
- `coverage_ratio = 0.0968` → `mean_abs_diff = 0.03657`
- `coverage_ratio = 0.2000` → `mean_abs_diff = 0.03242`

だった。

### 解釈
- 4096 でも傾向は崩れない
- RelayKV の主張は 1024 / 2048 だけでなく 4096 まで延ばせそう

## prompt type 比較

### 追加した prompt_type
- `repetitive`
- `prose`
- `structured`

### sweep 条件
- `seq_len = 1024, 4096`
- `hot_window = 128, 256`
- `block_size = 128, 256`
- `top_k = 1, 2, 3`
- `layer_idx = 27`
- `prompt_type = 3種類`

### 4096 / layer 27 で見えたこと
3種類すべてで

- coverage_ratio が上がるほど mean_abs_diff が下がる
- prompt を変えても coverage-driven trend は維持される

ことを確認した。

### prompt ごとの差
- `structured` が最も良い
- `prose` が中間
- `repetitive` がやや厳しい

という傾向が出た。

### 作成した図
- `relaykv_prompt_types_4096.png`

### 解釈
- RelayKV の傾向は単一の repetitive prompt に閉じていない
- prompt style によって絶対誤差は多少変わるが、基本傾向は安定している
